## 1. 사용자 여정 감정 곡선

### 전체 여정 개요

```
감정 ↑
     │
 기쁨 │                                              ★ 완주
  5  │        ★ 첫 가입                    ★ 첫 지출 승인
     │       /  \                         /
  4  │      /    \    ★ 첫 투표          /
     │     /      \   /    \            /
  3  │    /        ▽        \          /
     │   /       잔액 부족    \        /
  2  │  /                      \      /
     │ /                        \    /
  1  │/                          \  /
     │                            ▽
 불안 │                         권한 박탈
  0  ├────────────────────────────────────────────→ 시간
     가입     첫달    2-3개월   6개월    1년
```

## 2. 감정 상태 정의

| 감정 | 수치 | 설명 | 대표 상황 |
|------|------|------|----------|
| **기쁨/성취** | 5 | 목표 달성, 보상 획득 | 가입 완료, 완주, 승인 |
| **기대/흥분** | 4 | 새로운 시작, 참여 | 탐색, 첫 투표 |
| **평온/신뢰** | 3 | 안정적 사용 | 일상 사용 |
| **불안/걱정** | 2 | 불확실성, 비용 | 결제, 잔액 부족 |
| **좌절/스트레스** | 1 | 문제 발생 | 권한 박탈, 에러 |

## 3. 시나리오별 감정 흐름

### S1: 검색 진입 → 첫 챌린지 가입

```
감정
  5 │                                    ★ 가입 완료
  4 │ ★ 검색     ★ 상세 확인            /
  3 │  \       /  \                    /
  2 │   \     /    ▽ 결제 고민        /
  1 │    \   /                       /
  0 ├─────▽─────────────────────────────→
    검색  회원가입  챌린지확인  결제  완료
```

**UX 대응**:

| 감정 저점 | UX 대응 |
|----------|---------|
| 회원가입 번거로움 | 소셜 로그인, 최소 필드 |
| 결제 고민 | Trust Triangle 노출, 환불 정책 명시 |

---

### M2: 권한 박탈 → 복구

```
감정
  5 │                              ★ 권한 복구
  4 │
  3 │ ─ 평소 사용 ─
  2 │                   /─────────/
  1 │ ▼ 권한 박탈     /
  0 ├────────────────────────────────→
    평소    보증금사용  알림확인   충전   복구
```

**UX 대응**:

| 감정 저점 | UX 대응 |
|----------|---------|
| 권한 박탈 충격 | 명확한 사유 안내, 복구 방법 제시 |
| 복구까지 불안 | 진행 상황 표시, "60일 이내" 데드라인 명시 |

---

### L5: 챌린지 해산

```
감정
  5 │                              ★ 분배 완료
  4 │                            /
  3 │ ─ 운영 ─                  /
  2 │          ▼ 해산 결정    /
  1 │            \____________/
  0 ├────────────────────────────────→
    운영    해산결정  투표대기  전원동의  분배
```

**UX 대응**:

| 감정 저점 | UX 대응 |
|----------|---------|
| 해산 결정 무게감 | 충분한 안내, 최종 확인 모달 |
| 투표 대기 긴장 | 실시간 현황, 남은 인원 표시 |

## 4. 감정별 UX 대응 전략

### 4.1 긍정 감정 강화

| 감정 | 상황 | UX 표현 |
|------|------|--------|
| **기쁨** | 가입 완료 | Confetti 애니메이션 🎉 |
| **성취** | 첫 투표 완료 | Badge 획득 알림 |
| **자부심** | 1년 완주 | 인증 마크 + SNS 공유 |
| **소속감** | 첫 댓글 수신 | 하이라이트 알림 |

### 4.2 부정 감정 완화

| 감정 | 상황 | UX 표현 |
|------|------|--------|
| **불안** | 결제 | 환불 정책 명시, Trust Badge |
| **걱정** | 잔액 부족 | 명확한 금액 + 충전 CTA |
| **스트레스** | 권한 박탈 | 복구 가이드, 데드라인 표시 |
| **혼란** | 복잡한 플로우 | 진행률 인디케이터, 툴팁 |

## 5. 피크-엔드 법칙 적용

> 사용자는 경험의 **피크(최고점)**와 **엔드(마지막)**를 기억한다.

### 핵심 피크 모먼트

| 시점 | 피크 유형 | UX 강화 |
|------|----------|---------|
| 첫 가입 완료 | 긍정 피크 | 환영 애니메이션 + 온보딩 |
| 첫 투표 승인 | 참여 피크 | 결과 하이라이트 |
| 1년 완주 | 성취 피크 | Confetti + 인증 마크 |

### 피해야 할 네거티브 피크

| 시점 | 리스크 | UX 완화 |
|------|--------|---------|
| 결제 실패 | 이탈 | 즉시 재시도, 명확한 에러 |
| 권한 박탈 | 분노 | 복구 가능 강조, 사유 명시 |
| 해산 거부 | 좌절 | 재논의 안내, 피드백 채널 |

## 6. 감정 기반 마이크로카피

### 결제 관련

| 상황 | 일반 카피 | 감정 카피 |
|------|----------|----------|
| 결제 전 | "결제하기" | "안전하게 시작하기" |
| 결제 후 | "완료" | "🎉 환영합니다!" |
| 실패 | "실패" | "다시 시도해보세요. 금액은 청구되지 않았어요." |

### 권한 관련

| 상황 | 일반 카피 | 감정 카피 |
|------|----------|----------|
| 박탈 | "권한 없음" | "잠시 권한이 제한되었어요. 충전하면 바로 복구!" |
| 복구 | "복구됨" | "🎉 다시 돌아오셨네요!" |

### 투표 관련

| 상황 | 일반 카피 | 감정 카피 |
|------|----------|----------|
| 투표 요청 | "투표가 있습니다" | "여러분의 의견이 필요해요!" |
| 승인 | "승인됨" | "👍 멤버들이 동의했어요" |
| 거절 | "거절됨" | "아쉽지만 다음에 다시 논의해봐요" |
