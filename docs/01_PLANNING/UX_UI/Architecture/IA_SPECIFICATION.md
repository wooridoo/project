## 천재적 사고 공식 적용 분석

### 1. 문제 재정의 알고리즘 (Problem Redefinition)

**PR = P₀ × T(θ) × S(φ) × M(ψ)**

기존 IA의 문제를 다각도로 재정의:

| 변환 | 적용 | 결과 |
|------|------|------|
| **T(180°)** 반대 관점 | 기능 중심 → 사용자 여정 중심 | SNS 경험 → 신뢰 구축 → 금융 통합 |
| **S(0.5x)** 범위 축소 | 68개 화면 → 핵심 40개 화면 | Demo Day 필수 기능만 집중 |
| **M(+1)** 상위 개념 | "모임 상세" → "커뮤니티 홈" | 기능이 아닌 공간에 진입하는 경험 |

### 2. 복잡성 해결 매트릭스 (Complexity Solution)

**CS = det|M| × Σ[Si/Ci] × ∏[Ii]**

| 하위 시스템 | 기존 문제 | 해결책 |
|------------|----------|--------|
| Phase 순서 | SNS가 마지막 | SNS를 1순위로 재배치 |
| 어카운트 | 단순 충전 메뉴 | 가용/락 잔액 시스템 명시 |
| 가입 플로우 | 단순 신청 | 보증금 락 포함 2개월치 결제 |
| 모임 구조 | 대시보드 중심 | 피드(SNS) 중심으로 전환 |

## 1. 설계 원칙

### 1.1 핵심 가치 정렬

> **"소모임 + 토스 = 우리두"**

```
┌─────────────────────────────────────────────────────────────┐
│                    사용자 여정 (User Journey)                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   [1] 커뮤니티      [2] 신뢰 구축       [3] 금융 통합         │
│   ───────────      ───────────        ───────────          │
│   SNS 피드         선충전 락          투표 승인             │
│   댓글/좋아요      보증금 시스템       장부 투명화           │
│   공지사항         Django 분석        Recharts 차트         │
│                                                             │
│   "소모임처럼"      "먹튀 걱정 없이"    "토스처럼"            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 IA 설계 원칙

1. **SNS-First**: 피드가 모임의 중심 (1순위)
2. **Trust by Design**: 보증금/투표가 자연스럽게 녹아듦
3. **Mobile-First**: 모든 화면은 모바일 우선 설계
4. **Progressive Disclosure**: 복잡한 금융 정보는 단계적 노출
5. **Demo Day 역산**: 8주 개발 일정에 맞춘 우선순위

### 1.3 Design System Mapping (v2.0)

> **Type-Safe Implementation**: 개발 시 아래 토큰 매핑을 엄수하세요.

| UI Component | Token / Style | Usage Rule |
|--------------|---------------|------------|
| **Brand Identity** | `sys.color.primary` (#E9481E) | CTA, Active State, Logo |
| **Financial Text** | `sys.typescale.financial` | 금액 표시 시 필수 (Tabular Nums) |
| **Brix Badge** | `custom.brix.{level}` | 🍯(Gold), 🍇(Purple), 🍊(Mandarin) 등급 표시 |
| **Status Badge** | `sys.color.status.{state}` | Active(Orange), Suspended(Red) |
| **Card (Toss-like)** | `sys.shape.corner.large` (24px) | 메인 컨텐츠 컨테이너 |

### 4.3 어카운트 (Account) ⭐ NEW

**핵심 개념: 우리두 어카운트**

> 💡 **용어 정의**
> - **크레딧**: 우리두 가상머니
> - **보증금 락**: 챌린지 가입 시 미납 충당용으로 잠긴 크레딧
> - **오픈**: 챌린지 공동 자금 (서포트가 모인 금액)

```
┌─────────────────────────────────────────────────────────────┐
│                    우리두 어카운트                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   총 크레딧: ₩500,000  (Financial Text 적용)                     │
│   ├── 가용 크레딧: ₩300,000 (사용 가능)                       │
│   └── 보증금 락: ₩200,000 (챌린지별 잠금)                     │
│                                                             │
│   ┌─────────────┐  ┌─────────────┐                         │
│   │   충전하기   │  │   출금하기   │                         │
│   └─────────────┘  └─────────────┘                         │
│                                                             │
│   보증금 락 상세:                                            │
│   ├── 책벌레들 (서포트 ₩100,000) → 락: ₩100,000              │
│   └── 영화광들 (서포트 ₩100,000) → 락: ₩100,000              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 4.3.1 어카운트 대시보드

| 화면 | Type | Route | 설명 |
|------|------|-------|------|
| 잔액 카드 | Section | `/mypage` | 가용/락 잔액 분리 표시 |
| 락 상세 | BottomSheet | `/mypage` | 모임별 보증금 현황 |
| 거래 내역 | Page | `/mypage/history` | 충전/차감/락 히스토리 |

**2026 트렌드 적용:**
- Animated Counter: 잔액 변화 시 숫자 애니메이션
- Progress Ring: 락 비율 시각화
- Glassmorphism: 잔액 카드에 유리 효과

#### 4.3.2 충전

| 화면 | Type | Route | 설명 |
|------|------|-------|------|
| 충전 금액 선택 | Modal | `/charge` | 프리셋 + 직접 입력 |
| 결제 수단 | Modal | `/charge` | 토스페이 (MVP) |
| 충전 완료 | Toast | - | 성공 알림 + 잔액 갱신 |

**스펙:**
- 최소 충전: 10,000원
- 최대 충전: 1,000,000원
- 수수료: 없음

#### 4.3.3 출금

| 화면 | Type | Route | 설명 |
|------|------|-------|------|
| 출금 금액 입력 | Modal | `/withdraw` | 가용 잔액 한도 표시 |
| 출금 계좌 | Modal | `/withdraw` | 계좌 선택/등록 |
| 출금 완료 | Toast | - | 1-3일 소요 안내 |

#### 4.3.4 보증금 락 해제 플로우 ⭐ 개념 재정의

> 💡 **핵심 개념 변경**
> - **완주 ≠ 보증금 반환**: 완주는 1년 연속 운영 시 인증 마크 부여
> - **보증금 락 해제 조건**: 정상 탈퇴 시만 해제 (미납 없을 때)
> - **보증금 = 예치금**: 미납 시 서포트 충당용 안전장치
